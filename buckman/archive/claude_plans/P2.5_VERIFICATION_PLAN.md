# P2.5: Historical Data Archive & Ballpark Validation

## Overview

**Purpose:** Extract 2022-2023 data from PDF reports, create a structured multi-year baseline, and build a "ballpark check" module that catches gross errors before full regression testing.

**Rationale (Senior GW Modeler / USGS Best Practice):**
- Validation should happen EARLY, not late
- Historical envelope defines "reasonable" before we test for "exact"
- Every flagged value requires documented analyst disposition before release

**Location:** `validation/ballpark_check.py`

**Purpose:** Quick sanity check that runs in <5 seconds before full regression

**Key Functions:**

    FLAG if: outside historical min/max by >30%
    PASS if: within historical envelope
    """

def check_cumulative_monotonic(year_n_cumulatives: dict, bounds: dict) -> CheckResult:
    """
    HARD FAIL if: cumulative for year N < cumulative for year N-1
    This is a physics violation - can't undelete water
    """

def check_depletion_ratio(year_n_depletion: float, year_n_pumping: float,
                          reach: str, bounds: dict) -> CheckResult:
    """
    HARD FAIL if: ratio > 1.0 (depletion cannot exceed pumping)
    HARD FAIL if: ratio < 0 (negative depletion)

## Flagged Items / Uncertainties

1. **PDF extraction quality** - Tables in PDFs may have OCR artifacts. Plan for manual verification pass.

2. **2022 report format** - May differ from 2023/2024 format. Need to examine PDF structure.

3. **Well ID consistency** - Verify well naming is consistent across years (BW-1 vs BW1 vs Buckman 1).

4. **Threshold values** - The 30%, 50%, 0.85 correlation thresholds are starting points. May need tuning based on first run.

5. **Zero pumping wells** - Some wells may legitimately go offline. Need logic to distinguish "offline" from "missing data."
